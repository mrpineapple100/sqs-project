Glossar
=======

+------------------------+----------------------------------------------------------------------------------------------------------------------------------------+
| **Begriff**            | **Definition**                                                                                                                        |
+========================+========================================================================================================================================+
| **Spring Boot**        | Ein Java-basiertes Open-Source-Framework, das die schnelle Entwicklung produktionsreifer Anwendungen ermöglicht.                      |
|                        | Im Projekt wurde Java 17 und die neueste Spring Boot Version verwendet.                                                               |
+------------------------+----------------------------------------------------------------------------------------------------------------------------------------+
| **React**              | Eine JavaScript-Bibliothek zur Erstellung von Benutzeroberflächen. React wurde mit Vite verwendet, um das Frontend performant zu bauen.|
+------------------------+----------------------------------------------------------------------------------------------------------------------------------------+
| **Schichtenmodell**    | Ein Architekturmuster, das die Anwendung in Präsentation, Geschäftslogik und Datenzugriff unterteilt, um Wartbarkeit zu erhöhen.     |
+------------------------+----------------------------------------------------------------------------------------------------------------------------------------+
| **PostgreSQL**         | Ein relationales Open-Source-Datenbankmanagementsystem, das im Projekt zur Speicherung von Usern, Pokémon und Albumeinträgen dient.   |
+------------------------+----------------------------------------------------------------------------------------------------------------------------------------+
| **Backend-First-Ansatz**| Ein Entwicklungsansatz, bei dem zunächst das Backend entwickelt wird, um eine stabile API-Grundlage für das gesamte System zu schaffen.|
+------------------------+----------------------------------------------------------------------------------------------------------------------------------------+
| **Testing**            | Der Prozess der Prüfung der Funktionsfähigkeit der Anwendung. Im Projekt wurden JUnit, Integrationstests und Playwright eingesetzt.   |
+------------------------+----------------------------------------------------------------------------------------------------------------------------------------+
| **Docker**             | Eine Plattform zur Containerisierung von Anwendungen. Alle Komponenten des Projekts wurden dockerisiert für konsistente Umgebung.     |
+------------------------+----------------------------------------------------------------------------------------------------------------------------------------+
| **Docker Compose**     | Ein Tool zur Orchestrierung mehrerer Container. Es wird verwendet, um Backend, Frontend und Datenbank gemeinsam zu starten.           |
+------------------------+----------------------------------------------------------------------------------------------------------------------------------------+
| **GitHub Actions**     | Automatisierungs-Tool in GitHub, das zur Ausführung von Build-, Test- und Analysejobs bei jedem Commit verwendet wird.                |
+------------------------+----------------------------------------------------------------------------------------------------------------------------------------+
| **Qualitätssicherung** | Maßnahmen zur Sicherstellung der Funktionalität und Stabilität des Systems, z. B. durch Tests, Linting und SonarCloud-Analyse.        |
+------------------------+----------------------------------------------------------------------------------------------------------------------------------------+
| **SonarCloud**         | Eine Cloud-Plattform zur statischen Codeanalyse. Integriert in GitHub Actions, um Codequalität und Testabdeckung zu überwachen.       |
+------------------------+----------------------------------------------------------------------------------------------------------------------------------------+
| **Playwright**         | Ein Framework für End-to-End-Tests im Frontend. Wird verwendet, um Nutzerinteraktionen im Browser zu automatisieren und zu prüfen.    |
+------------------------+----------------------------------------------------------------------------------------------------------------------------------------+
| **OpenAPI**            | Eine Spezifikation zur Beschreibung REST-basierter APIs. Im Projekt wird sie für automatische Dokumentation der Backend-Endpunkte genutzt.|
+------------------------+----------------------------------------------------------------------------------------------------------------------------------------+
